## 一、项目概述

### 1.1 项目背景

随着高校信息化建设的不断推进，传统的纸质成绩管理方式已无法满足现代教学管理的需求。本系统旨在构建一个基于B/S架构的高校成绩管理系统，实现学生成绩的在线录入、查询、统计分析等功能，提高教学管理效率。

### 1.2 系统目标

- 实现多角色（管理员、教师、学生）分级权限管理
- 支持专业、班级、课程、成绩的全生命周期管理
- 提供在线选课、成绩录入、成绩查询等核心业务功能
- 具备响应式布局，适配多终端访问
- 支持Docker一键部署

### 1.3 技术选型

| 技术 | 版本 | 用途 |
|------|------|------|
| Spring Boot | 4.0.2 | 后端框架 |
| MyBatis-Plus | 3.5.16 | ORM框架 |
| MySQL | 8.0 | 数据库 |
| Vue 3 | 3.4 | 前端框架 |
| Element Plus | 2.6 | UI组件库 |
| Vite | 5.x | 前端构建工具 |
| Docker | 24.x | 容器化部署 |

```java
// AuthInterceptor.java 核心逻辑
if ("admin".equals(role)) return true;  // 管理员可访问所有接口
if ("teacher".equals(role)) {
    // 教师可访问: /teacher/*, /dashboard, /classes, /courses, /scores
}
if ("student".equals(role)) {
    // 学生可访问: /student/*, /dashboard, /course-selections
}
```

#### 4.2.2 路由权限守卫

```javascript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  if (!userStore.token) return next('/login')
  if (to.meta.roles && !to.meta.roles.includes(userStore.role))
    return next('/dashboard')
  next()
})
```

## 六、总结

### 6.1 技术亮点

1. **Spring Boot 4 + MyBatis-Plus 3.5.16**：采用最新技术栈，性能优越
2. **RBAC权限控制**：基于拦截器的角色权限管理，安全性高
3. **Docker容器化部署**：一键部署，环境一致性好
4. **响应式设计**：适配PC、平板、手机多终端
5. **小米设计风格**：现代化UI，用户体验优秀

### 6.2 系统特色

- 三角色分权管理，满足高校实际业务需求
- 在线选课功能，支持选课/退课操作
- 教师可自主发布课程
- 成绩统计分析，可视化展示
- 完整的测试数据，覆盖核心业务流程
